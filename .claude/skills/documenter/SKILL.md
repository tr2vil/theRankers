# /documenter - 문서화 담당

## 역할
theRankers의 문서화 담당. CLAUDE.md(AI 개발 가이드)와 README.md(사용자 가이드)를 최신 상태로 유지한다.

## 문서 역할 구분

| 문서 | 대상 | 목적 | 톤 |
|------|------|------|-----|
| **CLAUDE.md** | Claude Code (AI) | 개발 컨텍스트 - 아키텍처, 구조, 패턴, 규칙 | 기술적, 정확한 레퍼런스 |
| **README.md** | 사람 | 프로젝트 소개 - 기능, 설치, 사용법 | 친절한 가이드 |

## 문서화 필요 판단

### CLAUDE.md 업데이트 대상
- [ ] 새 모델 (models/) 추가/변경
- [ ] 새 API 엔드포인트 추가
- [ ] 새 서비스 (services/) 추가
- [ ] 수집기 추가/변경 (collector/)
- [ ] 스코어링/랭킹 로직 변경
- [ ] Celery 태스크 추가
- [ ] 데이터 파이프라인 변경
- [ ] Docker 서비스 변경
- [ ] 개발 규칙 변경

### README.md 업데이트 대상
- [ ] 주요 기능 구현 완료
- [ ] 기술 스택 변경
- [ ] 환경 변수 추가
- [ ] 설치/실행 방법 변경

### 문서화 불필요
- 버그 수정 (기존 동작 복원)
- UI 미세 조정
- 리팩토링 (인터페이스 변경 없음)
- 테스트 코드 추가

## 업데이트 원칙
1. 기존 구조와 포맷 유지
2. 관련 섹션만 업데이트
3. 실제 코드를 읽고 확인 후 작성 (추측 금지)
4. 기존 항목과 상세 수준 맞추기
